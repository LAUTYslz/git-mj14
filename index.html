Areas de GIT

Workking Directory

El Area de trabajo corresponde a la carpeta donde creamos el repositorio y basicamente vamos a crear, editar o eliminar archivos y subcarpetas. En muy pocas ocasines utilizaremos la consola y solamente necesitamos implementar el comando de GIT que inicializa el repositorio.

Staging Area(INDEX)

El area de control de datos corresponde a un archivo dentro de la carpeta .git que gestiona que archivos le realizamos cambios. Tambien se encarga de realizar las capturas de codigo para su posterior confirmacion cuando hayamos finalizado con el trabajo correspondiente.

Repository (local/remote)

El almacen de cambios corresponde a una estructura de carpetas dentro de la carpeta .git que contiene las confirmaciones realizadas en archivos BLOB (Binary Large Object). Se utilizan principalmente como referencia para comparar, visualizar y modificar el contenido de nuestro proyecto.
